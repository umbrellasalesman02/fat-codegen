# Generated port architecture: Structural Source, Projection Layer, and Progressive Field Curation

We will build the FileMaker port through deterministic vertical slices driven by `source/filemaker/funkis-authoring-tool/fm-xml-export-exploder-output/` as a **Structural Source** (tables/relationships/scripts/value lists metadata), while treating generated outputs as committed **Generated Packages** in `packages/` and keeping boundary contracts minimal in `packages/shared`. Runtime record retrieval will be handled by a hand-written **Projection Layer** (for example via OData) that curates only needed fields, and follows **Progressive Field Curation** (new fields are added only when a failing scenario/contract/script dependency requires them). We chose this over mirror-all-fields ingestion or domain-first waterfall phases because it preserves app/package decoupling, keeps scope intentional in a legacy-heavy schema, and makes regeneration/review deterministic and test-driven.
